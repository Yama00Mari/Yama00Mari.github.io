Study 4: The Number of Contributor (NoC) Determinator

The Number of Contributor (NoC) Determinator is software used to determine the number of contributors to a DNA sample. The tool can be used to analyze 2, 3, and 4-person samples. The software is written in java and uses machine learning to determine NoC. The machine learning algorithm implemented is Random Forest. The software uses the Random Forest algorithm from the weka interface to conduct the analysis. Weka is an open-source software developed by the University of Waikato. The Weka interface has different modules. These modules include Exploration, Experimentation, Knowledge Flow, Work Bench, just to name a few. In this project, the Exploration module of Weka was used. The NoC software results from Study 3(Mass Study). Thus, the model specifications described there are used to generate the model. 

Specifications of the software

1.  Required inputs: To analyze a sample, the software needs to be supplied with a sample file, a mass, and a model. Two of the inputs (sample file and mass) are provided by the user, while the model is determined by the software.  







2. Output file: A output file is created at the end of the analysis. The file contains a summary of the analysis. It provides information about the models used for the analysis and the final predicted NoC of the sample.    

Description of the software

The GUI interface of the software has a total of two pages (the analysis page and the final page). The analysis page (Figure 1) provides tools (buttons, text fields) that interact with the user. The final page (Figure 2) on the hand is shown at the end of the analysis to denote the end of the analysis. The final page also tells the user the name of the output file. The software has a total of 3 classes (GUI.java, Analyze.java, and Results.java ). The GUI.java class is responsible for creating the analysis page, the Analyze.java class is responsible for analyzing the sample, while Results.java class is responsible for creating the final page. A demonstration of how to use the software is shown below. 
Sample file: This is the file that contains the sample to be analyzed. This file needs to be in an arff format. It should also have a total of 355 attributes.
Mass: This is the mass of the sample in picogram (pg).
Model: this refers to the model used to analyze the sample. It is mainly based on the mass of the sample and the ratio of the contributors. The model is pulled up from predefined models (Study 3: Mass Study).

Limitations of the software

  1. Input files: The current version of the software can only analyze samples that are saved in arff format, in compliance with the Weka input requirement.
  2. The software can only analyze samples with NoC of 2-4.
  3.  Only one sample can be analyzed at a time.  

Future work  

    1.      Input Files: Make the software more universal so that it can accept inputs in txt, csv, json, etc.
    2.      Increase the NoC range of the software to accommodate samples with NoC of 1 and 5, and maybe more. 
    3.      Test the software on other DNA databases to see how robust the models are. 
    4.      Implement an automation protocol so that multiple samples can be analyzed at the same time. 
    5.      Experiment on ways of preprocessing the training samples to obtain higher accuracies. 

Conclusion 

NoC Determinator provides a graphical interface that is easy to use, fast and reliable. It can be used to determine the NoC of samples with different masses and different number of contributors.
